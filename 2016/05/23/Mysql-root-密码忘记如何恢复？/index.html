Centos Mysql root 密码忘记如何恢复？

发表于 2016-05-23   |  

Mysql 用户密码丢失，如何找回？或者清空？或者重置修改？

显然最直接有效的还是重置修改。

忘记了mysql 用户密码甚至是root的用户密码，导致不能用客户端连接数据库，苦恼。
如何拯救root？请看下文：
1.停止mysql服务，使用下面方法启动mysql

service mysql stop

mysqld_safe –skip-grant-tables &  #安全模式启动
mysqld_safe --skip-grant-tables --skip-networking &   #提高安全性，可以禁止远程连接

2.连接到mysql，不需要输入密码，直接回车

mysql -uroot –p  # mysql -p

3.重置root密码

update user set password=PASSWORD("abcdef") where user='root';
#Mysql5.7   
update mysql.user set authentication_string=password('123qwe') where user='root' and Host = 'localhost';

mysql-5.7.x mysql.user表里没有password字段了，取而代之的是authentication_string

4.重启Mysql

service mysql restart

5.赋权

但此时操作似乎功能不完全，还要alter user…

mysql> alter user 'root'@'localhost' identified by '123';

这样也可以：

mysql> set password for 'root'@'localhost'=password('123');

官方文档称自mysql_5.6后，首次启动会在/root/目录下产生.mysql_secret密码文件。

Centos root密码重置

这个技能据说许多公司都当作面试题，可见这事有多常见。

首先确定你拥有这台机器的所有权，能够重启。
1.Centos6 Booting时5s内按下任意键
2.Centos6 GRUB 画面时按e键，Centos7 GRUB2(由init初始化换成了systemd)启动时也按e键
3.Centos6 此时出现的新菜单里选Kernel/vmlinuz-2.xxx,再按下e键,在rlgb quiet行后加“空格”并键入”single”，或者直接输入数字的“1”并敲下Enter。
Centos7 出现编辑selected item,在kernel /vmlinuz-3.xxx 的行尾添加 init=/bin/sh
4.Centos6 此时按b(boot)键,进入单用户模式，随后就可以在sh-3.2#下passwd root 修改密码，改完reboot就ok。
Centos7 按Ctrl+x启动到shell单用户模式。
5.Centos7 需要挂载根分区为可读写模式

mount -o remount,rw
passwd

ps：如果开启selinux，根分区须有autorelabel文件，没有建之 touch /.autorelabel
全路径命令重启 exec /sbin/init or exec /sbin/reboot
