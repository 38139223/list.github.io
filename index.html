Mysql root 密码忘记如何恢复？

发表于 2016-05-23   |  

Mysql 用户密码丢失，如何找回？或者清空？或者重置修改？

显然最直接有效的还是重置修改。

忘记了mysql 用户密码甚至是root的用户密码，导致不能用客户端连接数据库，苦恼。
根据下面的方法，可以在任何linux发行版下重置root密码。centos mysql 忘记密码，ubuntu mysql 忘记密码,redhat mysql 忘记密码
1.停止mysql服务，使用下面方法启动mysql

service mysql stop

mysqld_safe –skip-grant-tables &  #安全模式启动
mysqld_safe --skip-grant-tables --skip-networking &   #提高安全性，可以禁止远程连接

2.连接到mysql，不需要输入密码，直接回车

mysql -uroot –p  # mysql -p

3.重置root密码

update user set password=PASSWORD("abcdef") where user='root';
#Mysql5.7   
update mysql.user set authentication_string=password('123qwe') where user='root' and Host = 'localhost';

mysql-5.7.x mysql.user表里没有password字段了，取而代之的是authentication_string

4.重启Mysql

service mysql restart

5.赋权

但此时操作似乎功能不完全，还要alter user…

mysql> alter user ‘root’@’localhost’ identified by ‘123’;
这样也可以：

mysql> set password for ‘root’@’localhost’=password(‘123’);
重点给大家介绍下mysql 5.7 root密码修改

MySQL管理者密码设置或修改：

依据官方说明5.6以后版本，第一次启动时会在root目录下生产一个随机密码，文件名.mysql_secret。

[root@bright ~]# cat /root/.mysql_secret

:Jj+FTiqvyrF
[root@bright ~]# cd /usr/local/mysql/bin/
[root@bright bin]# ./mysqladmin -u root -h localhost password ‘123456’ -p
Enter password: #此行输入.mysql_secret里第二行内容

mysqladmin: [Warning] Using a password on the command line interface can be insecure.
Warning: Since password will be sent to server in plain text, use ssl connection to ensure password safety.
官方的方式，笔者无论是否使用–skip-grant-tables启动mysql都测试失败，亲们可以测试：

shell>mysql -uroot -p’password’ #password即.mysql_secret里的密码
mysql>SET PASSWORD = PASSWORD(‘newpasswd’);

旧版本，安装后ROOT无密码，按如下操作：

方法一：

shell>service mysqld stop #停止mysql服务
shell>mysqld_safe –skip-grant-tables & #以不启用grant-tables模式启动mysql
shell>mysql -uroot -p #输入命令回车进入，出现输入密码提示直接回车。
mysql>use mysql;
mysql>update user set password=PASSWORD(“123456”)where user=”root”; #更改密码为 newpassord
mysql>flush privileges; #更新权限
mysql>quit #退出
方法二：

shell>service mysqld stop #停止mysql服务
shell>mysqld_safe –skip-grant-tables & #以不启用grant-tables模式启动mysql
shell>mysql -uroot -p #输入命令回车进入，出现输入密码提示直接回车。

CodeIgniter中数据库查询的实现

发表于 2016-05-17   |  

上文说完了CodeIgniter和数据库建立连接的设置，这一篇我们说说如何在CI框架里完成对数据库的查询操作。

基本查询

你经常用到的query函数，在做读类型的查询（select）时，query()函数将返回一个为对象类型的结果集；当执行写类型的操作（INSERT\DELETE\UPDATE）时，query函数则简单地返回TRUE/FALSE。你可以将query()函数返回的结果赋值给一个变量。

$query = $this->db->query('NEW QUERY');

简单化查询

simple_query函数是$this->db->query()的简化版，简单到只是提交一个查询，simple_query()函数返回数据库驱动器的”execute”方法的返回值。

if ($this->db->simple_query('NEW QUERY'))
{
    echo "Success!"
}
else
{
    echo "Query failed!"
}

指定数据库前缀

$this->db->set_dbprefix('newprefix');
$this->db->dbprefix('table_name');

你设定或者修改了数据库前缀参数，把它加在SQL语句的表名前面。

保护标识符

尽管使用查询构造器会自动保护标识符，有人还是愿意使用手工语句：

$this->db->protect_identifiers('table_name');

protect_identifiers(‘table_name’,TRUE);函数的第二个参数作用是启用你在数据库配置文件中定义的dbprefix参数。

转义查询

CodeIgniter提供了三种方法实现转义。

  1. $this->db->escape()函数仅转义字符串类型，加单引号。

    $sql = “INSERT INTO table (title) VALUES(“.$this->db->escape($title).”)”;

  2. $this->db->escape_str()函数忽略数据类型。

    $sql = “INSERT INTO table (title) VALUES(“.$this->db->escape_str($title).”)”;

  3. $this->db->escape_like_str()函数用于处理LIKE语句中的字符串。

    $search = ‘20% raise’;
    $sql = “SELECT id FROM table WHERE column LIKE ‘%” .
    $this->db->escape_like_str($search).”%’”;

查询绑定

查询绑定可以简化你的查询语法，它通过系统自动的为你将各个查询组装在一起：

$sql = "SELECT * FROM some_table WHERE id = ? AND status = ? AND author = ?";
$this->db->query($sql,array(3,'live','Rick'));

查询语句中的问号将会自动被第二个参数位置的数组的相应的值替代。
也可以使用数组的数组进行绑定，里面的数组会被转换成IN语句的集合：
查询绑定时，如果参数是整数，要用(int)转换一下，要不然也会被加上单引号

$sql = "SELECT * FROM some_table WHERE id IN ? AND status = ? AND author = ?";
$this->db->query($sql,array(array(3,6),'live','Rick'));

上面的例子被转换成这样的SQL语句：

SELECT * FROM some_table WHERE id IN(3,6) AND status = 'live' AND author = 'Rick'

查询绑定将值自动转义，安全查询。

错误处理

$this->db->error()方法可以得到一个包含错误代码和错误消息的数组。

if (! $this->db->simple_query('SELECT `example_field` FROM `example_table`'))
    {
        $error = $this->db->error(); //Has keys 'code' and 'message'
    }

重建Mysql从库

发表于 2016-05-17   |  

作为一个DBA，做烦心的事情莫过于主从数据不一致，但谁让你赶上了，废话不说，赶紧的把正事办了。

第一步，在负载均衡器中摘掉故障slave，登录slave，查看slave状态，确定无流量后关掉slave。

show slave status\G;
show processlist;
stop slave;

第二步，确定该slave的master状态。

show slave status\G;

第三步，清除slave的Databases，binlog

drop database XXXX;
show master status\G;
purge binary logs to binlog.000xxx;

or

reset master;

第四步，从master导出数据

nohup mysqldump -h192.168.88.66  -uroot -pabcdef -P 3306 -y --default-character-set=utf8 --master-data=2 --single-transaction -A > /data/db.sql  &

ps：

  1. –master-data=2：该选项将二进制日志的位置和文件名写入到输出中。该选项要求有RELOAD权限，并且必须启用二进制日志，值为1时CHANGE MASTER语句会被dump输出，值为2时该语句被输出为SQL注释。由于导出用户和复制用户不一致，故此处值为2.
  2. -y 不导出表空间。
  3. -A 导出所有的database
  4. –single-transaction 不要锁表。

  5. –default-character-set=utf8 字符集参数

    第五步，数据导入slave

    #nohup mysql -proot@’abcdef’ –default-character-set=utf8 < db.sql &

    or
    source db.sql;
    stop slave;
    reset slave;

第六步，从数据里查找binlog位置，对齐。

grep 'CHANGE MASTER TO MASTER_LOG_FILE' db.sql |more
CHANGE MASTER TO MASTER_HOST='192.168.88.66',MASTER_USER='sync',MASTER_PASSWORD='9999999',MASTER_LOG_FILE='mysql-bin.000027', MASTER_LOG_POS=237322923;

第七步，开启slave，恢复主从复制。

start slave;
show slave status\G;

观察服务状态。如果你运气好，至此，主从复制已经恢复，怎么样，去喝一杯吧。

Shell定时从备份中恢复一个数据库

发表于 2016-05-17   |  

非常简单的脚本。

#!/bin/bash
DAY=`date -d "1 day ago" +%Y%m%d`
EXDAY=`date -d "3 day ago" +%Y%m%d`
HOSTNAME="127.0.0.1" #hostname
PORT="3306"
USERNAME="root"
PASSWORD="123456"
EX_DBNAME=${EXDAY}_product
DB_NAME=${DAY}_product
drop_exdb="drop database ${EX_DBNAME};"
create_db_sql="create database ${DB_NAME}; "
use_db="use ${DB_NAME};"
mysql -h${HOSTNAME} -P${PORT} -u${USERNAME} -p${PASSWORD} -e "${drop_exdb}"
mysql -h${HOSTNAME} -P${PORT} -u${USERNAME} -p${PASSWORD} -e "${create_db_sql}"
mysql -h${HOSTNAME} -P${PORT} -u${USERNAME} -p${PASSWORD} -e "${use_db}"
cd /data/product/
mysql -h${HOSTNAME} -P${PORT} -u${USERNAME} -p${PASSWORD} ${DB_NAME} < ${DAY}_product.sql
rm -rf /data/product/${DAY}_product.sql

其中${DAY}_product.sql是每天晚上都会定时备份的库文件。

CodeIgniter如何连接到mysql数据库

发表于 2016-05-16   |  

连接数据库

CodeIgniter连接数据库，可以自动或者手动。

  1. 自动连接，每个页面加载时自动实例化数据库类。application/config/autoload.php文件中的library数据里添加database

    $autoload[‘libraries’] = array(‘database’);

  2. 手动连接，在有需要的函数里手工添加代码连接数据库，或写道类的构造函数里，让整个类访问。

    $this->load->database();

如果 database()函数没有指定第一个参数，它将使用数据库配置文件中指定的组连接。

$this->load->database('group_name');

连接一个完全手动指定的数据库，也可以穿一个数组参数；

$config['hostname'] = 'localhost';
$config['username'] = 'myusername';
$config['password'] = 'mypassword';
$config['database'] = 'mydatabase';
$config['dbdriver'] = 'mysqli';
$config['dbprefix'] = '';
$config['pconnect'] = FALSE;
$config['db_debug'] = TRUE;
$config['cache_on'] = FALSE;
$config['cachedir'] = '';
$config['char_set'] = 'utf8';
$config['dbcollat'] = 'utf8_general_ci';
$this->load->database($config);

对于 PDO 驱动，你应该使用 $config[‘dsn’] 取代 ‘hostname’ 和 ‘database’ 参数：

$config['dsn'] = 'mysql:host=localhost;dbname=mydatabase';

当用 DSN 字符串连接时，要覆盖默认配置，可以像添加查询字符串一样添加配置变量。

$dsn = 'dbdriver://username:password@hostname/database?char_set=utf8&dbcollat=utf8_general_ci&cache_on=true&cachedir=/path/to/cache';
$this->load->database($dsn);

连接到多个数据库

项目中经常用到连接多个数据库：

$DB1 = $this->load->database('group_one', TRUE);
$DB2 = $this->load->database('group_two', TRUE);

第一个参数group_two等是你要连接的组名称，第二个参数 TRUE 表示函数将返回数据库对象。
当使用这种方法连接数据库时，你将通过你的对象名来执行数据库命令：

$DB1->query();
$DB2->result();
etc...
与之前的方法对比一下
$this->db->query();
$this->db->result();
etc...

ps:
如果只是想切换到同一个连接不同的数据库，那就没必要创建独立的数据库配置：

$this->db->db_select($database2_name);

重新连接，保持连接有效

当你的php代码处理任务复杂（处理图片等）时，很有可能会超过数据库的超时值，此时可以考虑在执行后续查询之前先调用reconnect()方法向数据库发送ping命令，这样可以保持连接有效或重新建立连接。

$this->db->reconnect();

手动关闭连接

CodeIgniter可以智能关闭数据库连接，你也可以选择手动关闭。

$this->db->close();

Shell遍历指定目录并只保留日期最近的10个文件

发表于 2016-05-12   |  

将文件所在目录列表存到一个数组

#!/bin/bash
selfname=`basename $0`   #获得脚本自身文件名
x=0         #数组索引初始
for filename in *;
do         #循环当前目录下的文件
   if [ "$selfname" = "$filename" ];then
      continue   #本文件跳过
   else
      array[$x]=$filename   #文件名给数组索引为x的元素
      let "x+=1"      #改变索引指针，步长1
   fi
done
echo ${array[@]}   #打印数组，以便检查程序运行情况
echo $x

遍历指定目录的子目录和文件，保留最新的10个文件，其他删除

#!/bin/bash
W=/var/logs/Service/job
Dlist=(`ls $W`)
#echo ${Dlist[*]}
for i in ${Dlist[*]}
do
cd $W/$i/logs;ls -r |awk '{if(NR>11){print $0}}'|xargs rm -rf
done

CodeIgniter 数据库配置

无论是测试还是生产环境，你都需要一个配置文件<默认位置application/config/database.php>来保存数据库连接信息(用户名、密码、数据库名等),配置存放在一个多维数组里。

$db['default'] = array(
'dsn'   => '',
'hostname' => 'localhost',
'username' => 'root',
'password' => '',
'database' => 'database_name',
'dbdriver' => 'mysqli',
'dbprefix' => '',
'pconnect' => TRUE,
'db_debug' => TRUE,
'cache_on' => FALSE,
'cachedir' => '',
'char_set' => 'utf8',
'dbcollat' => 'utf8_general_ci',
'swap_pre' => '',
'encrypt' => FALSE,
'compress' => FALSE,
'stricton' => FALSE,
'failover' => array()
);

有些数据库驱动（例如：PDO，PostgreSQL，Oracle，ODBC）可能需要提供完整的 DSN 字符串。在这种情况下，你需要使用 ‘dsn’ 配置参数，就好像使用该驱动的 PHP 原生扩展一样。例如:

// PDO
$db['default']['dsn'] = 'pgsql:host=localhost;port=5432;dbname=database_name';

// Oracle
$db['default']['dsn'] = '//localhost/XE';

如果你没有指定DSN字符串，会自动生成，如果缺少某些配置，CodeIgniter将从其他配置项中存在的配置自动附加。
当主数据库由于某些原因无法连接时，你还可以配置故障转移（failover）。可以像下面这样为一个连接配置故障转移:

$db['default']['failover'] = array(
    array(
        'hostname' => 'localhost1',
        'username' => '',
        'password' => '',
        'database' => '',
        'dbdriver' => 'mysqli',
        'dbprefix' => '',
        'pconnect' => TRUE,
        'db_debug' => TRUE,
        'cache_on' => FALSE,
        'cachedir' => '',
        'char_set' => 'utf8',
        'dbcollat' => 'utf8_general_ci',
        'swap_pre' => '',
        'encrypt' => FALSE,
        'compress' => FALSE,
        'stricton' => FALSE
    ),
    array(
        'hostname' => 'localhost2',
        'username' => '',
        'password' => '',
        'database' => '',
        'dbdriver' => 'mysqli',
        'dbprefix' => '',
        'pconnect' => TRUE,
        'db_debug' => TRUE,
        'cache_on' => FALSE,
        'cachedir' => '',
        'char_set' => 'utf8',
        'dbcollat' => 'utf8_general_ci',
        'swap_pre' => '',
        'encrypt' => FALSE,
        'compress' => FALSE,
        'stricton' => FALSE
    )
);

可以指定任意多个故障转移。
使用多维数组可以随意存储多个连接值（开发、测试、生产等），每个环境设定独立连接组且可切换，以“test”环境为例：

$db['test'] = array(
'dsn'   => '',
'hostname' => 'localhost',
'username' => 'root',
'password' => '',
'database' => 'database_name',
'dbdriver' => 'mysqli',
'dbprefix' => '',
'pconnect' => TRUE,
'db_debug' => TRUE,
'cache_on' => FALSE,
'cachedir' => '',
'char_set' => 'utf8',
'dbcollat' => 'utf8_general_ci',
'swap_pre' => '',
'compress' => FALSE,
'encrypt' => FALSE,
'stricton' => FALSE,
'failover' => array()
);

然后，在配置文件中设置$active_group变量，告诉系统使用“test”组:

$active_group = 'test';

实际生产应用中，设定发布程序会根据你的选择，修改该值。

Codeigniter初见

发表于 2016-05-11   |  

CodeIgniter初见

Q：如何安装？
A：非常简单，[官网](http://codeigniter.org.cn)下载后，上传至测试网站目录下，
浏览器打开http://localhost/就可以了。

CodeIgniter目录结构

application
-cache 缓存
-config 配置文件
-controllers 控制器
-core 核心库扩展目录
-errors 错误
-helpers自定义辅助函数文件夹
-hooks钩子文件目录
-language 本地语言包
-libraries自定义库文件，类文件
-logs 日志
-models 模型
-third_party 引用第三方包
-views 视图
system
index.php

  • 1、controllers控制器
    控制器就是一个类文件,须继承CI_Controller.
    命名规范：
  • A、类名的首字母应该大写。多个词之间用下划线连接。
  • B、类中的方法名称小写，多个词之间下划线连接。

  • C、类文件名使用小写，文件名与类名保持一致。
    Eg：china.php

    <?php
    class China extends CI_controller{
    public function beijing(){

     echo "Welcome to beijing!";
}

    }
    ?>

URL规则-pathinfo规则，分隔符”/“
http://b-gg.com/citest/index.php/china/beijing
http://example.com/[controller-class]/[controller-method]/[arguments]

  • 2、views视图
    view常用来做模板，后缀可以是html、php。

  • 3、controllers调用views

    public function beijing(){
    $this->load->view(‘beijing.html’);
    }

    简单地将views文件展示出来，实际上我们需要在controllers控制器中获取到数据，然后
    分配到视图中展示。

    public function beijing(){
    $data[‘title’] = ‘中国首都’;
    $data[‘content’] = ‘北京是中华人民共和国的首都。’；
    $this->load->view(‘beijing.html’ .$data);
    }
    相应地视图文件应该这么写：

    <body>
 <h2><?php echo $title; ?></h2>
 <p><?php echo $content; ?></p>
</body>


    需要注意的是，输出直接使用$title,而不是$data[‘title’],这是因为框架extract函数将数组变量导入到当前的符号表，可以直接使用键名作为变量使用。

    4、要点：
    • 1、框架开发和二次开发的区别
      CI是提供的是基础功能和项目结构，项目的功能需要你自己完成。
      ecshop等开源项目的二次开发则是另一回事。
    • 2、MVC
      M:model模型，操作数据库
      V:view视图，展示数据。
      C:controller控制器，接受请求-调用模型工作和获取数据，分配到视图展示。
